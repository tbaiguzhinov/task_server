Создать API сервис, соответствующий следующим пунктам:
 1) Сервис должен иметь следующую архитектуру: server (принимает запросы, передает задачу в worker и возвращает
    ответ клиенту), worker (принимает задачу от сервера и возвращает результат), БД (для логирования ошибок).
    Каждый элемент сервиса должен работать в отдельном контейнере, общение между
    сервером и воркером выполняется при помощи брокера сообщений.
 2) Клиент отправляет на сервер excel файл, описывающий продажи за некоторый период, с колонками Date (YYYY-MM-DD),
    Sales (float), в которой пропущены некоторые дни. Требуется вернуть excel файл с недостающими данными
    (способ восстановления пропусков обосновать).
 3) При возникновении ошибок на уровне обработки таблицы (неверные данные в таблице, пустая таблица и т.д.),
    необходимо обработать их и вернуть ответ клиенту с описанием ошибки; записать
    время возникновения ошибки, uid, описание ошибки.
 4) Сформировать docker-compose.yaml и документацию по деплою сервиса.
 5) В качестве UI можно использовать Swagger или удобный фреймворк для фронтенда
